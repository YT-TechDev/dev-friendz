# Contributing

Dev Friendz uses GitHub Issues and Milestones as the canonical task tracker.

## Workflow

1. Check existing Issues and Milestones.
2. Use or request a scoped Issue before substantial work.
3. Create a focused feature branch.
4. Make the smallest coherent change.
5. Avoid unrelated refactoring.
6. Run validation required by the Issue.
7. Open a Pull Request against `main`.
8. Include validation evidence, known limitations, and remaining device checks.
9. Complete real-device checks when the change affects ScriptWidget rendering or runtime behavior.
10. Merge only after required checks pass.

## Documentation and communication

Public OSS documentation and contribution content should be in English.

Never include AI session-sharing URLs in Issues, Pull Requests, commits, or documentation.

## Security and privacy

Never commit secrets, tokens, PATs, API secrets, credentials, private account data, private repository data, or personal configuration. Do not place secrets in tracked configuration files.

See [`SECURITY.md`](SECURITY.md) for the security policy.

## License compatibility

Contributions should be compatible with the repository's MIT License. See [`LICENSE`](LICENSE).

This project does not currently imply a contributor governance body, automated release system, or maintainer team.
